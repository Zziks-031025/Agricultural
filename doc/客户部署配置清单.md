# 农产品溯源系统 - 客户部署配置清单

本文档面向在客户设备上进行完整部署的操作人员，涵盖所有需要修改的配置项及操作步骤。

## 二、步骤一：启动 Ganache

1. 安装并打开 Ganache 桌面应用
2. 点击 **QUICKSTART (ETHEREUM)**
3. 确认页面顶部 RPC Server 显示为：

   ```
   HTTP://127.0.0.1:8545
   ```

4. 在 **ACCOUNTS** 标签页，复制第一个账户的地址（格式 `0x...`），填入上方表格"Ganache 账户地址"

> Ganache 必须在系统运行期间始终保持开启状态。若关闭后重启，链上数据会丢失，需重新部署合约并更新配置。

---

## 三、步骤二：部署智能合约并获取合约地址

### 3.1 打开 Remix IDE

浏览器访问：https://remix.ethereum.org/

### 3.2 导入合约文件

1. 左侧 **File Explorer** 面板，点击新建文件图标
2. 文件名填写 `Traceability.sol`
3. 将项目 `05_SmartContract/Traceability.sol` 的全部内容粘贴进去

### 3.3 编译合约

1. 左侧切换到 **Solidity Compiler** 标签（图标为 S 形）
2. Compiler 版本选择 `0.8.x`（任意 0.8 版本）
3. 点击 **Compile Traceability.sol**
4. 编译成功后左侧图标变为绿色

### 3.4 连接 Ganache

1. 左侧切换到 **Deploy & Run Transactions** 标签（以太坊菱形图标）
2. **ENVIRONMENT** 下拉框选择 `Custom - External Http Provider`
3. 弹窗中填入：

   ```
   http://127.0.0.1:8545
   ```

4. 点击 **OK**，连接成功后 **ACCOUNT** 下拉框自动加载 Ganache 账户列表
5. 确认 **ACCOUNT** 选中的是步骤二中记录的第一个账户地址

### 3.5 部署合约

1. **CONTRACT** 下拉框确认选中 `Traceability - Traceability.sol`
2. 点击橙色 **Deploy** 按钮
3. 部署成功后，页面下方 **Deployed Contracts** 区域出现合约条目
4. 点击合约条目左侧的复制图标，复制合约地址
5. 将复制的地址填入上方表格"智能合约地址"

### 3.6 获取合约 ABI（如需更新）

项目已提供 `05_SmartContract/contract_abi.json`，正常情况无需重新获取。

若合约代码有修改，需重新获取 ABI：
1. 在 **Solidity Compiler** 标签页，编译完成后
2. 点击 **ABI** 按钮（位于 Compile 按钮下方）复制内容
3. 替换项目中 `05_SmartContract/contract_abi.json` 的内容

## 四、步骤三：初始化数据库

### 4.1 创建数据库

打开命令行，执行：

```bash
mysql -u root -p
```

输入 MySQL 密码后，执行：

```sql
CREATE DATABASE agricultural DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

### 4.2 导入表结构及初始数据

```bash
mysql -u root -p agricultural < 01_Database/agricultural_database.sql
```

## 五、步骤四：配置后端

### 5.1 修改配置文件

打开文件：`02_Backend/src/main/resources/application.yml`

需要修改的配置项如下：

```yaml
spring:
  datasource:
    # 如果 MySQL 密码不是 123456，修改此处
    password: ${DB_PASSWORD:123456}

blockchain:
  # Ganache RPC 地址，默认不需要修改
  rpc-url: http://127.0.0.1:8545

  # 填入步骤二获得的智能合约地址，必须加引号
  contract-address: "0x填入合约地址"

  # 填入步骤一记录的 Ganache 第一个账户地址，必须加引号
  account-address: "0x填入账户地址"
```

### 5.2 启动后端（正常模式）

```bash
cd 02_Backend
mvn clean compile
mvn spring-boot:run
```

启动成功标志：控制台出现

```
Tomcat started on port(s): 8888
```

### 5.3 注入演示数据（可选）

如需向数据库注入演示用的溯源数据（100 个肉鸡批次，约 2000 条溯源记录，并同步上链），使用 `seed` Profile 启动后端：

```bash
cd 02_Backend
mvn spring-boot:run "-Dspring-boot.run.profiles=seed"
```

**注意事项：**

- 执行前必须确保 Ganache 已启动，且 `application.yml` 中合约地址和账户地址已正确填写，因为注入过程会真实调用智能合约上链
- 注入过程耗时较长（约 2~3 分钟），控制台会逐批输出进度，等待出现以下日志后表示完成：

  ```
  DataSeeder 完成，成功注入 100/100 个批次
  ```

- 注入完成后后端会自动停止，需重新以正常模式启动（步骤 5.2）
- 该操作只需执行一次，重复执行会因为 traceId 已存在而导致上链失败（数据库记录仍会重复写入，建议只执行一次）

## 六、步骤五：配置并启动 PC 前端

### 6.1 开发环境启动

```bash
cd 03_Frontend_Web
npm install
npm run dev
```

访问 http://localhost:8080，默认账号：

| 角色 | 用户名 | 密码 |
|------|--------|------|
| 平台管理员 | admin | admin123 |

### 6.2 生产环境构建（部署到服务器时）

修改 `03_Frontend_Web/.env.production`：

```
VUE_APP_BASE_API = 'http://客户设备IP:8888'
VUE_APP_BACKEND_URL = 'http://客户设备IP:8888'
```

然后执行构建：

```bash
cd 03_Frontend_Web
npm run build
```

将生成的 `dist/` 目录部署到 Nginx 或其他 Web 服务器。

## 七、步骤六：配置微信小程序

### 7.1 需要修改的配置文件

小程序所有需要修改的配置集中在以下两个文件：

---

#### 文件 1：`04_Frontend_MiniApp/config.js`（必须修改）

此文件控制小程序连接的后端地址，是最关键的配置文件。

**场景 A：局域网内调试（手机与电脑在同一 WiFi）**

```javascript
const ENV = 'dev'   // 保持 dev

const ENV_CONFIG = {
  dev: {
    // 将此处改为客户设备的局域网 IP + 后端端口
    // 查看方式：cmd 运行 ipconfig，找到 IPv4 地址
    API_BASE_URL: 'http://192.168.x.x:8888',   // 改为实际 IP
  },
  ...
}
```

**场景 B：正式发布（后端部署到公网服务器，需 HTTPS）**

```javascript
const ENV = 'prod'   // 改为 prod

const ENV_CONFIG = {
  ...
  prod: {
    // 改为实际的后端域名，微信小程序正式版必须使用 HTTPS
    API_BASE_URL: 'https://your-domain.com',
  }
}
```

---

#### 文件 2：`04_Frontend_MiniApp/project.config.json`（发布时需修改）

此文件包含小程序的 AppID，决定小程序归属于哪个微信公众平台账号。

当前 AppID：`wxf24401df8316ca97`

**如果客户有自己的小程序账号，需要修改：**

```json
{
  "appid": "wx客户的AppID"
}
```

> AppID 获取方式：登录 https://mp.weixin.qq.com/ → 开发 → 开发管理 → 开发设置 → AppID

---

### 7.2 导入项目到微信开发者工具

1. 打开微信开发者工具
2. 点击 **导入项目**
3. 项目目录选择 `04_Frontend_MiniApp`
4. AppID 填入对应账号的 AppID（或选择测试号）
5. 点击 **导入**

### 7.3 开发调试设置（局域网调试必须开启）

1. 工具栏点击 **详情**
2. 切换到 **本地设置** 标签
3. 勾选 **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**

> 此设置仅在开发者工具中生效，不影响正式发布版本。正式发布版本必须在微信公众平台配置合法域名。

### 7.4 真机调试

1. 手机与电脑连接同一 WiFi
2. 确认 `config.js` 中 `API_BASE_URL` 已改为电脑局域网 IP
3. 微信开发者工具顶部点击 **预览** 或 **真机调试**
4. 手机微信扫码即可

### 7.5 正式发布流程

1. 确认 `config.js` 中 `ENV` 已改为 `'prod'`，`API_BASE_URL` 为 HTTPS 域名
2. 在微信公众平台 **开发 → 开发设置 → 服务器域名** 中添加后端域名
3. 微信开发者工具顶部点击 **上传**，填写版本号和备注
4. 登录 https://mp.weixin.qq.com/ → 版本管理 → 提交审核

## 九、常见问题

| 现象 | 原因 | 解决方案 |
|------|------|---------|
| 后端启动报 `Access denied for user 'root'` | MySQL 密码错误 | 修改 `application.yml` 中 `datasource.password` |
| 后端启动报 `Connection refused: 127.0.0.1:8545` | Ganache 未启动 | 先启动 Ganache |
| 后端启动报 `Contract call reverted` | 合约地址配置错误 | 确认 `contract-address` 与 Remix 部署地址一致 |
| 小程序报 `request:fail` | IP 地址错误或未关闭域名校验 | 检查 `config.js` 中 IP，并勾选"不校验合法域名" |
| 小程序图片不显示 | `IMAGE_BASE_URL` 未正确拼接 | 确认 `config.js` 中 `API_BASE_URL` 填写正确 |
| 上链操作全部失败 | Ganache 重启导致合约丢失 | 重新部署合约，更新 `contract-address` |
| 小程序正式版接口失败 | 未配置服务器域名或未使用 HTTPS | 在微信公众平台添加合法域名 |
