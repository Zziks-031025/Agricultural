- 1. # 基于区块链的农产品溯源模版系统需求文档

     ## 1. 文档头部

     - **文档编号**：QT-2026012903
     - **创建日期**：2026年01月29日
     - **版本号**：1.0
     - **项目名称**：基于区块链的农产品溯源模版系统

     ## 2. 项目概述

     本项目基于Spring Boot框架和以太坊区块链技术，构建一套农产品溯源模版系统。项目核心在于建立适合不同农产品类型（以**肉鸡养殖**和**西红柿种植**为典型案例）的溯源模版，实现从初始化、生长管理、质检、加工到销售的全流程数字化记录。系统包含**溯源模版管理系统子平台**、**企业管理端**（PC）、**企业小程序端**和**用户小程序端**，通过智能合约实现数据的上链存证与查询验证，解决农产品溯源中的信任问题。

     ## 3. 项目目标

     1. **架构落地**：实现管理端、企业端（PC+小程序）、用户端（小程序）的多端协同架构。
     2. **模版化设计**：针对不同农产品（肉鸡/西红柿）设计标准化的溯源模版，涵盖初始化及生长过程。
     3. **三体协同**：支持**养殖/种植**、**宰杀/加工**、**检疫**三个实体角色的业务流转。
     4. **区块链存证**：利用以太坊智能合约记录关键环节哈希值，实现数据防篡改与可验证。

     ## 4. 系统架构

     - **后端**：Java (Spring Boot) + MyBatis + MySQL。
     - **前端**：Vue.js + Element UI (Web端)，微信小程序原生开发 (移动端)。
     - **区块链**：以太坊 (Ethereum) + Web3j + Solidity智能合约。

## 6. 其他说明

1. **业务逻辑规则（重点）**：系统必须实现**严格的权限隔离**。
   - **注册时**：企业选择类型（如：我是养殖户）。
   - **审核时**：平台管理员确认其类型正确。
   - **登录时**：系统根据类型下发菜单权限。养殖户**绝不能**看到“检疫申报”入口；检疫机构**绝不能**看到“创建养殖批次”入口。
2. **技术实现**：后端需使用 Web3j 与以太坊区块链进行交互，实现数据上链和查询验证功能。
3. **数据验证**：系统需通过 MessageID 在链上查询哈希值，并与本地数据库生成的哈希值进行对比，以验证数据是否被篡改。
